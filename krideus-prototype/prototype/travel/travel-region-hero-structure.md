# K.Rideus 여행 페이지 구조 제안 — 지역 선택형 히어로

작성일: 2026-06-09  
대상 경로: `k.rideus.net` 여행/Place 페이지 개편안  
참고 구조: `https://tourtaxi.triseup.com/nations`

---

## 1. 논의 배경

TourTaxi의 `/nations` 페이지는 다음과 같은 구조를 가진다.

```text
상단 로고 / 언어 버튼
↓
지역 탭 메뉴
↓
메인 카피 + 한국 지도
↓
선택 지역명
↓
지역별 관광택시 카드/링크
↓
관광택시 소개 섹션
↓
푸터
```

이 구조의 핵심은 다음이다.

```text
지역 선택
→ 선택 지역 표시
→ 지역별 투어 링크 노출
```

다만 TourTaxi 구조를 그대로 K.Rideus에 가져오기에는 한계가 있다.

- 지도 중심 UI가 관광 포털/공공서비스처럼 보일 수 있음
- 지역 탭이 너무 상단에 있어 선택 결과가 바로 보이지 않음
- 지도와 카드 목록 사이의 연결감이 약함
- K.Rideus는 단순 관광택시 링크 모음이 아니라 여행 장소와 이동 서비스를 함께 연결해야 함

따라서 K.Rideus 여행 페이지에는 TourTaxi의 지역 선택 구조를 차용하되, 지도는 제거하고 **지역 선택형 히어로 + 지역별 콘텐츠 전환 구조**로 재해석하는 방향이 적합하다.

---

## 2. 최종 방향

K.Rideus 여행 페이지는 다음처럼 정의한다.

> 한국 여행 지역을 선택하면, 해당 지역의 분위기와 대표 여행 장소, 그리고 이동 가능한 서비스를 함께 보여주는 Place 탐색 허브.

또는 더 간단히:

> 지역과 유형으로 찾아보는 K.Rideus 여행 장소 컬렉션.

핵심 흐름은 다음이다.

```text
히어로에서 여행 페이지의 정체성 전달
↓
히어로 하단 지역 메뉴에서 지역 선택
↓
선택된 지역의 히어로 이미지/카피 변경
↓
바로 아래에서 해당 지역의 장소 컬렉션 확인
↓
이동 문의 CTA로 연결
```

## 3. 지도 제거 판단

### 3.1 지도 제거가 더 적합한 이유

TourTaxi는 전국 관광택시 지역 게이트 성격이 강하기 때문에 지도 UI가 자연스럽다.  
하지만 K.Rideus는 여행지 소개와 이동 서비스 연결을 동시에 해야 하므로, 지도보다 지역별 대표 이미지가 더 적합하다.

지도 대신 지역별 히어로 이미지를 사용하면 다음 장점이 있다.

1. **프리미엄 여행 브랜드 톤 강화**
   - 지도는 기능적이고 행정구역 중심으로 보일 수 있음
   - 이미지는 여행 감성과 목적지 욕구를 더 잘 전달함

2. **지역 분위기 전달력 강화**
   - 강원: 설산, 리조트, 동해
   - 서울: 도심, 호텔, 문화
   - 경기·인천: 테마파크, 쇼핑, 공항
   - 경상: 부산 해안, 경주 역사
   - 전라: 한옥, 정원, 미식
   - 제주: 해안도로, 오름, 섬 풍경

3. **모바일 대응 용이**
   - 지도 SVG는 모바일에서 터치 영역/가독성 문제가 생기기 쉬움
   - 지역 메뉴는 모바일 가로 스크롤 탭으로 처리 가능

4. **콘텐츠 부족 지역을 유연하게 관리 가능**
   - 지도는 전국이 모두 보이기 때문에 빈 지역이 도드라질 수 있음
   - 메뉴 방식은 현재 운영 가능한 지역만 노출하거나, 우선순위를 조정하기 쉬움

## 4. 지역 메뉴 위치

지역 메뉴는 히어로 상단이 아니라 **히어로 하단**에 둔다.

### 4.1 이유

TourTaxi의 경우 지역 탭이 상단에 있어, 사용자가 지역을 선택해도 결과가 아래에 있어 변화가 즉시 보이지 않는다.

K.Rideus에서는 다음 흐름이 더 자연스럽다.

```text
히어로 카피/이미지 확인
↓
지역 메뉴 선택
↓
바로 아래에서 선택 결과 확인
```

즉, 지역 메뉴는 히어로와 결과 콘텐츠 사이의 브릿지 역할을 해야 한다.

### 4.2 권장 배치

```text
[Hero]
- 페이지 메시지
- 선택 지역 오버라인
- 선택 지역명
- 선택 지역 태그라인
- 설명
- CTA
- 지역 대표 이미지

[Hero Bottom Region Menu]
서울 / 경기·인천 / 강원 / 경상 / 전라 / 제주

[Selected Region Content]
- 지역 요약
- 유형 필터
- 대표 장소 카드
- 이동 예시
```

### 4.3 Sticky 동작 옵션

초기 위치는 히어로 하단에 두되, 사용자가 아래로 스크롤하면 상단에 sticky로 붙는 방식도 고려할 수 있다.

```text
초기: 히어로 하단
스크롤 이후: 상단 sticky region nav
```

단, 처음부터 상단에 고정하지는 않는다.  
초기에는 반드시 히어로 하단에서 선택 결과와 가까운 위치에 있어야 한다.

## 5. 지역 메뉴와 키워드 처리

지역 메뉴는 간결하게 지역명만 제공한다.

```text
서울
경기·인천
강원
경상
전라
제주
```

지역별 키워드는 메뉴 안에 넣지 않고, **히어로 섹션의 오버라인 또는 태그라인**으로 제공한다.

### 5.1 이유

메뉴 안에 키워드를 넣으면 메뉴가 무거워지고 모바일에서 복잡해진다.

예를 들어 다음 구조는 정보량이 많다.

```text
강원
리조트 · 자연

경기·인천
쇼핑 · 테마파크
```

대신 메뉴는 선택 기능만 담당하고, 선택된 지역의 감성/맥락은 히어로에서 보여준다.

```text
RESORT · NATURE · EAST COAST
강원
설산과 리조트, 동해 바다를 따라 떠나는 여행
```

역할 분리는 다음과 같다.

```text
지역 메뉴: 선택 기능
히어로 오버라인/태그라인: 선택 지역의 분위기 전달
하단 콘텐츠: 선택 결과와 장소 목록 표시
```

## 6. 지역 구분

기존 `수도권`은 `서울`과 `경기·인천`으로 분리한다.

### 6.1 최종 지역 메뉴안

```text
서울
경기·인천
강원
경상
전라
제주
```

### 6.2 서울과 경기·인천을 분리하는 이유

서울과 경기·인천은 같은 수도권이지만 사용자 목적과 서비스 맥락이 다르다.

#### 서울

서울은 여행지이자 한국 여행 이동의 출발점이다.

주요 맥락:

```text
도심 여행
호텔 픽업
공연/이벤트 이동
쇼핑/미식
외국인 게스트 접대
서울 출발 근교 이동
```

#### 경기·인천

경기·인천은 서울 근교 목적지와 공항 연결 지역 성격이 강하다.

주요 맥락:

```text
테마파크
프리미엄 아울렛
인천공항
송도/영종도
근교 당일치기
가족 여행
기업/단체 이동
```

따라서 `수도권` 하나로 묶으면 메시지가 흐려질 수 있다.  
`서울`과 `경기·인천`으로 나누면 각 지역의 히어로 카피와 장소 카드 구성이 더 선명해진다.

## 8. 상세 섹션 설계

## S1. Region Hero

### 목적

- 여행 페이지의 정체성을 선언
- 선택된 지역의 분위기를 이미지와 카피로 전달
- 사용자가 지역 선택 이후 어떤 콘텐츠가 바뀌는지 인지하게 함

### 구성 요소

```text
페이지 라벨
지역별 오버라인
선택 지역명
지역별 태그라인
지역별 설명
CTA
지역 대표 이미지
```

### 기본 와이어

```text
TRAVEL PLACES

RESORT · NATURE · EAST COAST

강원

설산과 리조트, 동해 바다를 따라 떠나는 여행

스키 리조트부터 해안 도시, 로컬 투어까지.
K.Rideus가 강원 여행의 이동을 함께 연결합니다.

[장소 둘러보기] [이동 문의하기]

                         [강원 대표 이미지]
```

## S1-1. Hero Bottom Region Menu

### 목적

- 히어로와 결과 콘텐츠 사이에서 지역 선택 컨트롤 역할
- 선택 즉시 아래 콘텐츠가 바뀐다는 것을 직관적으로 전달

### 구성

```text
서울   경기·인천   강원   경상   전라   제주
```

### 디자인 후보

#### A. Pill Tab

```text
[서울] [경기·인천] [강원] [경상] [전라] [제주]
```

장점:
- 직관적
- 모바일 대응 쉬움
- 필터 UI로 인지하기 쉬움

#### B. Underline Tab

```text
서울   경기·인천   강원   경상   전라   제주
                ───
```

장점:
- 더 차분하고 프리미엄한 인상
- 버튼 느낌이 덜함

K.Rideus에는 `Pill Tab` 또는 `Underline Tab` 모두 가능하나, 전체 브랜드 톤이 프리미엄/에디토리얼이면 `Underline Tab`이 더 적합하다.

---

## S2. Selected Region Overview

### 목적

- 선택된 지역의 성격을 다시 요약
- 아래 장소 카드가 어떤 기준으로 나오는지 설명
- 히어로를 지나온 사용자에게 선택 결과를 명확히 보여줌

### 구성 예시

```text
강원 대표 장소

설산과 리조트, 동해 바다를 따라 떠나는 여행지.
스키 리조트부터 해안 도시, 관광택시까지 다양한 이동 목적지를 확인해보세요.

주요 유형
리조트 · 자연 · 관광택시

이동 옵션
셔틀 · 프라이빗 밴 · 단체 이동
```

### 주의

지역 선택 시 히어로가 바뀌더라도, 사용자가 아래 콘텐츠에 있을 때는 히어로 변화가 보이지 않을 수 있다.  
따라서 S2에서 지역 요약을 한 번 더 반복하는 것이 좋다.

## S3. Type Filter

### 목적

지역 선택 후, 장소를 유형으로 한 번 더 좁히는 기능.

### 기본 유형

```text
전체
리조트
쇼핑
테마파크
관광택시
시티투어
자연
```

지역별로 해당 없는 유형은 비활성 또는 미노출 처리할 수 있다.

예:

```text
강원: 전체 / 리조트 / 자연 / 관광택시
서울: 전체 / 시티투어 / 쇼핑 / 문화
경기·인천: 전체 / 테마파크 / 쇼핑 / 공항
제주: 전체 / 자연 / 관광택시 / 프라이빗 투어
```

### 목적

선택된 지역에 맞는 대표 여행 장소를 보여준다.

```text

장소명
지역 · 유형

이동 정보
CTA
```

### 카드 예시

```text
휘닉스 평창
강원 · 리조트

스키와 사계절 액티비티를 즐길 수 있는 평창 대표 리조트

서울 출발 약 2시간 30분
셔틀 · 프라이빗 밴

[장소 보기] [이동 문의]
```

```text
파주 프리미엄 아울렛
경기·인천 · 쇼핑

서울 근교에서 즐기는 프리미엄 쇼핑 여행

서울 출발 약 40분
프라이빗 이동 추천

[장소 보기] [이동 문의]
```

### 카드 설계 원칙

현재 데이터 기준으로는 상황별 추천이나 과한 큐레이션 문구보다 다음 정보가 중요하다.

```text
장소의 성격
지역
유형
이동 가능성
문의 연결
```

따라서 “당신에게 딱 맞는 추천” 같은 표현은 피하고, **K.Rideus가 이동과 연결 가능한 대표 여행 장소를 정리했다**는 톤이 적합하다.

## S5. Mobility Example

### 목적

K.Rideus가 단순 여행지 목록 사이트가 아니라, 목적지 이동을 연결하는 서비스임을 보여준다.

TourTaxi에는 없는 섹션이지만 K.Rideus에는 필요하다.

### 예시: 강원

```text
강원 리조트 여행은 이렇게 연결됩니다

서울 호텔 픽업
→ 휘닉스 평창 / 하이원 / 비발디파크
→ 숙소 체크인 또는 서울 복귀

추천 이동
프라이빗 밴 · 장비 적재 가능 · 가족/단체 이동
```

### 예시: 경기·인천

```text
서울 근교 쇼핑·테마파크 여행

서울 호텔 또는 인천공항 픽업
→ 에버랜드 / 파주 프리미엄 아울렛 / 여주 프리미엄 아울렛
→ 호텔 드롭 또는 다음 목적지 이동

추천 이동
프라이빗 차량 · 쇼핑 짐 보관 · 외국인 게스트 이동
```

## S6. Why K.Rideus Travel

### 목적

하단에서 K.Rideus 여행 이동 서비스의 장점을 설명한다.

TourTaxi의 하단 소개 섹션을 차용하되, 관광택시 중심이 아니라 K.Rideus 이동 경험 중심으로 재작성한다.

### 구성 예시

```text
좋은 장소를 더 편하게 만나는 방법

01. 장소 선택부터 이동까지
여행지는 고르기 쉽지만, 이동은 복잡할 수 있습니다.
K.Rideus는 목적지에 맞는 이동 방식을 함께 안내합니다.
```

## 11. 디자인 주의사항

### 11.1 이미지 품질

이 구조는 지도 대신 이미지를 중심으로 하기 때문에, 지역별 대표 이미지 품질이 매우 중요하다.

피해야 할 이미지:

```text
관광청 스톡처럼 보이는 이미지
해상도 낮은 이미지
색감이 제각각인 이미지
장소 정체성이 불분명한 이미지
텍스트와 겹쳤을 때 가독성이 떨어지는 이미지
```

권장 이미지 방향:

```text
서울: 도심, 호텔, 문화, 야경
경기·인천: 테마파크, 아울렛, 공항, 송도/영종도
강원: 설산, 리조트, 동해, 숲길
경상: 부산 해안, 경주 유적, 남쪽 도시
전라: 한옥, 정원, 미식, 남도 풍경
제주: 해안도로, 오름, 바다, 공항 이동
```

### 11.2 한영 혼합

오버라인은 영어, 태그라인은 한국어를 추천한다.

예:

```text
RESORT · NATURE · EAST COAST
강원
설산과 리조트, 동해 바다를 따라 떠나는 여행
```

이 조합은 세련된 인상과 한국어 이해도를 동시에 확보할 수 있다.

### 11.3 메뉴 정보량

지역 메뉴에는 키워드를 넣지 않는다.

```text
좋음:
서울 / 경기·인천 / 강원 / 경상 / 전라 / 제주

피함:
서울 - 도시·문화
경기·인천 - 쇼핑·테마파크·공항
강원 - 리조트·자연·동해
```

키워드는 히어로 오버라인/태그라인에서 처리한다.

### 11.4 선택과 결과의 거리

지역 메뉴 바로 아래에 선택 결과가 보여야 한다.

```text
히어로
↓
지역 메뉴
↓
선택 지역 요약
↓
장소 카드
```

TourTaxi처럼 지역 메뉴와 결과 콘텐츠 사이가 멀어지면 선택 변화 체감이 약해진다.

## 13. 최종 정리

TourTaxi의 구조에서 가져올 핵심은 `지도`가 아니라 다음 흐름이다.

```text
지역 선택
→ 지역별 콘텐츠 전환
→ 관련 목적지/서비스 연결
```

K.Rideus에서는 이를 다음처럼 재구성한다.

```text
지역 선택형 히어로
→ 히어로 하단 지역 메뉴
→ 선택 지역별 대표 이미지/카피
→ 지역별 장소 컬렉션
→ 이동 문의 CTA
```

최종 구조의 핵심 판단은 다음과 같다.

1. 지도는 제거한다.
2. 지역 메뉴는 히어로 하단에 둔다.
3. 지역 메뉴는 지역명만 간결하게 제공한다.
4. 지역별 키워드는 히어로 오버라인/태그라인으로 처리한다.
5. `수도권`은 `서울`과 `경기·인천`으로 분리한다.
6. 선택 결과는 메뉴 바로 아래에서 보여준다.
7. 장소 카드는 과한 추천보다 지역·유형·이동 가능성을 중심으로 구성한다.
8. K.Rideus의 차별점은 하단 Mobility Example과 CTA에서 분명히 보여준다.

한 줄 결론:

> K.Rideus 여행 페이지는 지도 기반 관광 포털이 아니라, **지역별 히어로 이미지와 장소 컬렉션을 통해 여행 목적지를 발견하고 이동 문의로 이어지는 프리미엄 Place 탐색 허브**로 설계하는 것이 적합하다.
