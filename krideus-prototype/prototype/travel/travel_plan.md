# 왜 서울과 경기/인천을 나누는 게 좋은가

## 1. 사용자의 목적이 다름

“서울”과 “경기/인천”은 같은 수도권이지만 여행 목적이 꽤 다릅니다.

### 서울
서울은 보통 이런 목적입니다.

```text
시티투어
호텔 픽업
공항 도착 후 도심 이동
쇼핑/미식
외국인 게스트 접대
공연/이벤트 연계
```

즉 서울은 “여행지”이면서 동시에 **출발지/허브** 역할이 큽니다.

### 경기/인천
경기/인천은 이런 목적입니다.

```text
테마파크
아울렛 쇼핑
공항 접근
근교 당일치기
가족 여행
기업/단체 이동
```

대표적으로:

```text
에버랜드
파주 프리미엄 아울렛
여주 프리미엄 아울렛
시흥 프리미엄 아울렛
인천공항
송도
영종도
```

이쪽은 서울과 다르게 **목적지형 이동**이 더 강합니다.

그래서 하나의 “수도권”으로 묶으면 메시지가 흐려질 수 있어요.

---

# 2. K.Rideus의 이동 서비스와도 잘 맞음

K.Rideus에서는 서울이 단순 여행지가 아니라 이동의 기준점일 가능성이 큽니다.

```text
서울 호텔 → 강원 리조트
서울 호텔 → 경기 테마파크
인천공항 → 서울
인천공항 → 강원
인천공항 → 경기 아울렛
```

이런 식으로요.

따라서 `서울`을 독립 메뉴로 두면, 나중에 서울 기반 이동/투어 콘텐츠를 확장하기 좋습니다.

---

# 3. 콘텐츠 운영도 더 유연함

`수도권` 하나로 묶으면 카드가 뒤섞입니다.

```text
롯데월드
에버랜드
파주 아울렛
여주 아울렛
인천공항
송도
서울 시티투어
```

이게 모두 한 섹션에 들어가면 다소 산만합니다.

반면 분리하면:

```text
서울
- 서울 시티투어
- 롯데월드
- 쇼핑/공연/도심 이동
- 호텔 픽업

경기/인천
- 에버랜드
- 파주 아울렛
- 여주 아울렛
- 시흥 아울렛
- 인천공항/송도/영종도
```

훨씬 정리됩니다.

# 서울을 넣을 때 주의할 점

서울은 “여행지”이기도 하지만 “출발지”이기도 해서, 콘텐츠 방향을 잘 잡아야 합니다.

서울 섹션이 단순히:

```text
서울 관광지 목록
```

이 되면 K.Rideus와 연결이 약할 수 있습니다.

그래서 서울은 이렇게 정의하는 게 좋습니다.

> 서울은 여행 목적지이자, 한국 여행 이동의 출발점이다.

즉 서울 카드에는 이런 성격을 넣을 수 있습니다.

```text
서울 시티투어
호텔 픽업
공연/이벤트 이동
도심 쇼핑
서울 출발 근교 이동
```

반면 경기/인천은:

> 서울에서 떠나는 근교 목적지와 공항 연결 지역

으로 정의하면 좋습니다.

# 경기/인천을 묶는 건 괜찮나?

네, 경기와 인천은 묶어도 괜찮아 보입니다.

이유는:

```text
1. 둘 다 서울 근교 이동권으로 인식됨
2. 인천은 공항/송도/영종도 중심이라 독립 여행 섹션으로 두기엔 초기 콘텐츠가 약할 수 있음
3. 경기 콘텐츠와 함께 묶으면 테마파크/쇼핑/공항이라는 명확한 축이 생김
```

다만 라벨은 살짝 고민해볼 수 있습니다.

## 라벨 후보

```text
경기/인천
```

가장 직관적입니다.

```text
경기·인천
```

더 깔끔하고 디자인적으로 좋습니다.

```text
서울 근교
```

감성적이지만 인천공항이 포함된다는 점이 덜 명확합니다.

```text
근교/공항
```

서비스 관점은 좋지만 지역명으로는 약합니다.

저는 메뉴 라벨은:

```text
경기·인천
```

을 추천합니다.

본문에서는:

```text
서울 근교와 공항을 잇는 목적지
```

라고 설명하면 됩니다.
