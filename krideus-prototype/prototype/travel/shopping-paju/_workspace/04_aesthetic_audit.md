# 파주 프리미엄 아울렛 상세 — 미적(Aesthetic) 비평

> 시니어 비주얼 디자이너 관점 · 시각만(기능·링크·실데이터·SEO·성능 제외)
> 근거: 데스크탑 1380px 렌더 4분할 + 소스 `index.html`(CSS 인라인) 토큰 대조
> 기준: taste-skill `soft-skill`(고급 비주얼) + `taste-skill`(편향 보정) — 단, 사용자 확정 원칙(그림자 자제/박스 안 박스 지양/오버라인 금지/4px 그리드)을 상위 규범으로 둠

## 종합 평점: 8.0 / 10

**한 줄 총평** — 크림 베이스 위에 코발트·바이올렛을 절제 있게 얹어 "프리미엄 쇼퍼 서비스"의 톤을 안정적으로 구현했다. 정보 위계와 여백 리듬은 프로덕션급이나, **히어로의 임팩트 부족**과 **컬러 배분이 본문 상단에 코발트로 쏠리고 바이올렛이 거의 안 보이는 불균형**, 그리고 **6개 서비스 카드의 3×2 균질 그리드**가 상위 점수를 막는다.

---

## 잘된 점 (구체 근거)

1. **컬러 베이스의 절제 — 크림 위 단일 액센트 원칙 준수** (part1~3 전반)
   `--bg:#FAF8F5` / `--surface:#FFFEFC` / `--card:#FFFFFF`로 따뜻한 3단 화이트를 깔고, 액션 색은 코발트(`#2C51DB`) 하나로 고정. soft-skill의 "Max 1 Accent" 규범에 부합하고, 순백(#FFF) 대신 웜 크림을 써서 저렴한 느낌을 피했다("NO Pure White/cheap base" 원칙). 면(surface)·여백·톤차로만 섹션을 가른 점도 사용자 확정 원칙(섹션 구분=면으로만)과 일치한다.

2. **타이포 위계의 명확한 3단 — 페이지 타이틀 / 섹션 타이틀 / 카드** (part1~2)
   `pd-title clamp(2rem,3.8vw,3rem)/800/-.03em` → `sec-title clamp(1.6rem,3vw,2.2rem)/700/-.03em` → 카드 `1.06rem/700`로 weight·size·tracking이 단계마다 분명히 떨어진다. 한글 본문에 `word-break:keep-all`을 걸어 어절 단위 줄바꿈을 보장해 가독성을 챙겼다. Outfit(영문 라벨·숫자) + Pretendard(한글 본문) 분업도 깔끔하다.

3. **sticky 예약폼의 정보 밀도와 여백 균형** (part1 우측)
   세그먼트(편도/왕복/풀데이) → 픽업/도착 → 날짜·시간 → 카운터 → 선택 차량 → 예상가 → CTA → 신뢰 칩의 수직 흐름이 위에서 아래로 자연스럽게 읽힌다. 입력 필드를 `border:none + 배경 var(--bg)`로 처리해 도드라진 1px 라인을 피한 것이 사용자 원칙(라인 비선호)과 정확히 맞는다.

4. **At a Glance 십자선 그리드 — 박스 없이 라인만** (part1 좌하단)
   6개 항목을 `gap:1px;background:var(--hair)`의 내부 십자선만으로 구분(`.pd-glance`). 카드 박스를 안 쓰고 좌우 폼 높이와 균형을 맞춘 영리한 선택. dt를 Outfit 대문자 마이크로 라벨(코발트), dd를 굵게, small을 ink-soft로 둔 3단 위계가 작은 공간에서도 또렷하다.

5. **Sample Day 타임라인의 절제된 디테일** (part2~3)
   바이올렛 도트 + `box-shadow:0 0 0 4px var(--private-soft)` 헤일로 + 2px 커넥터 라인으로, 과하지 않게 "프라이빗" 색을 처음으로 의미 있게 노출. 타임/노드/본문 3컬럼 그리드 정렬이 정확하다.

## 개선점 (임팩트 순)

### 1. [최우선] 히어로가 약하다 — 컨테이너 폭 제한 + 25vh 높이 + 카피 부재
- **위치/근거**: part1 최상단. `.phero{height:clamp(220px,25vh,300px);max-width:1280px;margin:0 auto;border-radius:0 0 32px 32px}`
- **무엇이 어색한가**: (a) 1380px 뷰포트에서 히어로가 1280px로 제한되어 **양옆에 크림 여백 띠**가 생기고, 풀폭 사진의 몰입감이 깨진다. (b) 높이가 화면의 1/4에 불과해 290여 개 브랜드 아울렛이라는 스케일이 사진으로 전달되지 않는다. (c) 사진 위에 **타이틀·카피가 전혀 없다** — scrim(상단 어둠 그라데이션)만 깔리고 텍스트는 사진 아래 본문으로 내려가, 첫인상에서 "어디로, 누구와"라는 후킹이 비어 있다. soft-skill "Standard Hero Paradigm"은 사진 위 비대칭 타이포를 요구한다.
- **구체 개선안**:
  - 풀폭 전환: `.phero{max-width:none;border-radius:0}` 로 풀블리드. 하단 라운드가 필요하면 사진은 풀폭, 콘텐츠 래퍼만 살짝 위로 겹치게(`margin-top:-32px;border-radius:32px 32px 0 0`).
  - 높이 상향: `height:clamp(320px,42vh,460px)`.
  - 사진 위 카피 1줄 추가: 좌하단 정렬로 `eyebrow(여행 · 쇼핑) + H1(신세계 파주 프리미엄 아울렛)`. scrim을 `linear-gradient(0deg,rgba(10,14,20,.62),transparent 55%)`로 하단 강화. (단, 사용자 "섹션 오버라인 금지"는 본문 섹션 타이틀 위 영문 오버라인에 한정 — 히어로 브레드크럼 성격 한 줄은 허용 범위. 우려되면 H1만.)

### 2. 컬러 배분 불균형 — 상단은 코발트 과잉, 바이올렛은 거의 실종
- **위치/근거**: part1 좌측 본문 + part1 우측 폼. At a Glance dt 6개 전부 코발트(`--accent`), svc-card 번호 코발트, fleet 베스트가 코발트, CTA·폼 전부 코발트. 반면 바이올렛(`--private`)은 폼 헤더의 작은 "프라이빗 기사" 배지(part1 우상단)와 추천 카드 태그, Sample Day 도트에서만 등장 — **"이 서비스 전체가 프라이빗 기사"라는 핵심 메시지가 색으로 거의 안 보인다.**
- **무엇이 어색한가**: 과제 정의의 "코발트(액션) + 바이올렛(프라이빗)" 역할 분담이 화면에서 9:1로 쏠려, 바이올렛이 "찍어둔 토큰"처럼만 존재한다. 동시에 At a Glance 6개 dt가 전부 같은 코발트라 라벨이 액션 컬러와 혼동된다.
- **구체 개선안**:
  - At a Glance dt 컬러를 `--accent`(코발트) → `--ink-faint`(`#97A0AC`) 또는 `--private`로 변경. 코발트는 "누를 수 있는 것(CTA·링크·선택)"에만 남겨 의미를 정화. 권장: dt는 중립 `--ink-faint`, "Chauffeur" 항목만 바이올렛으로 포인트.
  - "프라이빗 기사" 배지를 폼 헤더에서 살짝 키우거나(현재 height:24px → 26px, font .7rem → .74rem), pd-lead 아래에 바이올렛 도트 + "전 일정 프라이빗 기사 동행" 한 줄 라벨을 추가해 색의 의미를 본문 상단에서 한 번 더 못박는다.

### 3. 서비스 카드 — "3-column 균질 그리드" 6개가 단조롭다
- **위치/근거**: part2 "택시도, 셔틀도 아닌 이유" 섹션. `.svc-grid{grid-template-columns:repeat(3,1fr)}` × 6개, 각 카드가 `01~06 번호 + h3 + p`로 형태·크기 완전 동일.
- **무엇이 어색한가**: soft-skill·taste-skill 모두 "3 equal cards horizontally" 균질 피처 그리드를 AI-slop 신호로 본다. 6개가 똑같은 무게로 나열되어 **시선이 머물 1순위(기사 종일 대기·고정 요금)가 다른 4개와 구분되지 않는다.** 한 화면에서 6개가 동등하게 경쟁한다.
- **구체 개선안** (사용자 원칙 위배 없는 선에서):
  - 1·2번(핵심 차별점)을 상단 2컬럼 와이드로, 3~6번을 하단 4컬럼 또는 2×2 작은 항목으로 — 정보 위계를 레이아웃으로 표현. CSS: 첫 행 `grid-column:span 3`(2개), 둘째 행 4개. 또는
  - 최소 개입안: 6개 유지하되 1·2번 카드 `h3`를 `1.06rem→1.18rem`로, 번호를 시각적으로 키워(`.svc-card__no .78rem→.9rem`) 상단 2개에 무게를 실어 균질감 완화.
  - 카드별 미세 `max-width:38ch`가 이미 있어 우측 끝이 들쭉날쭉할 수 있으니, 시각적 정렬을 위해 동일 행 카드 텍스트 폭 통일 점검.

### 4. 히어로 라운드 vs 풀폭 섹션 — 라운드 모서리 일관성 결여
- **위치/근거**: 히어로만 `border-radius:0 0 32px 32px`(part1 상단). 그 외 fleet-card 20px, ocard `clamp(14px,1.2vw,18px)`, route-map 20px, bk `clamp(16px,1.4vw,22px)` 등 라운드 값이 14·18·20·22·32로 5종 분산.
- **무엇이 어색한가**: 라운드 토큰이 정리되지 않아 "기계 가공된 일관성"(soft-skill의 concentric radius 철학)이 깨진다. 특히 히어로 32px와 본문 카드 20px의 점프가 크다.
- **구체 개선안**: 라운드 스케일을 3단으로 토큰화 — `--r-sm:14px / --r-md:20px / --r-lg:28px`. 히어로 하단 32→28, 폼·맵·fleet은 20(md)로 수렴, 작은 칩·태그만 999px 유지. 4px 그리드 원칙에 맞춰 14·20·28(전부 미세하게 어긋나면 12·20·28로).

### 5. CTA 다크 섹션의 휑한 하단 여백
- **위치/근거**: part4. CTA(`.cta`)와 footer가 한 다크 면으로 이어지나, 캡처 기준 CTA 콘텐츠 아래로 **거대한 빈 다크 공간**이 보인다. 푸터는 `foot__brand` 단일 컬럼(로고+2줄)뿐이라 좌측만 차고 우측 절반이 비었다.
- **무엇이 어색한가**: 채널 칩(WhatsApp/WeChat/LINE/Kakao) 아래에서 푸터까지 호흡이 과도하게 비어, 페이지 끝맺음이 "미완성"처럼 느껴진다. 푸터가 좌측 정렬 단일 블록이라 우측 공백도 균형이 안 맞는다.
- **구체 개선안**:
  - CTA `padding-bottom`을 `clamp(48px,6vw,80px)` → `clamp(40px,5vw,64px)`로 축소.
  - 푸터에 우측 컬럼 추가(약식 사이트맵: 이동/여행/이벤트 + 연락처) 또는 `foot__in`을 `justify-content:space-between`이 살도록 우측 링크 블록 보강. 최소안: 푸터 브랜드 블록을 중앙 또는 좌우 2컬럼(브랜드 / 운영시간·연락)로 채워 우측 공백 제거.

### 6. 예상가/Fleet 숫자 타이포의 강조 부족
- **위치/근거**: part1 폼 예상가 `.bk__total b{font-size:1.4rem}`, fleet best가 `1.1rem 코발트`(part1 하단). 폼의 핵심 전환 포인트인 "예상 요금"이 1.4rem로, 페이지 타이틀(3rem) 대비 작고 코발트도 아닌 ink 기본색.
- **무엇이 어색한가**: 가격은 예약 페이지에서 의사결정의 핵심인데, 시선 끌기가 약하다. fleet 카드의 "풀데이" best 가격만 코발트인데 폼 합계는 검정이라 색 일관성도 흔들린다.
- **구체 개선안**: `.bk__total b` `1.4rem→1.6rem`, 숫자에 `color:var(--ink)` 유지하되 `₩`(.won)만 ink-faint로 더 죽여 숫자를 돋보이게. `font-variant-numeric:tabular-nums`는 이미 적용됨(굿). "부터" small은 현행 유지.

## 사소한 디테일 폴리시 목록

- **데모 잔재 이모지 제거**: `bk__done`의 "✓ 예약이 접수되었습니다", fleet CTA의 "✓ 선택된 차량" — 체크 이모지(✓)는 taste-skill 안티-이모지 규범 위반. SVG 체크 또는 마스크 아이콘으로 교체(이미 `bk__trust li::before`에 SVG 체크 마스크가 있으니 재사용).
- **fleet 고스트 태그의 "—"**: Solati 카드 `<span class="fleet-card__tag is-ghost">—</span>` — `visibility:hidden`이라 화면엔 안 보이나, 정렬 목적이면 빈 문자열이 깔끔. 텍스트 "—" 제거 권장.
- **글자수 클램프 정렬**: `svc-card max-width:38ch`, `pd-lead 54ch`, `pd-about p 60ch` 등 ch 단위가 섞여 있어 같은 컬럼 내 우측 끝이 미세하게 어긋날 수 있음. 동일 영역은 한 값으로 통일.
- **bk__seg "풀데이" 기본 선택 강조**: 세그먼트 on 상태가 `background:#fff + box-shadow 1px 2px`(미세 그림자) — 사용자 그림자 자제 원칙상, 그림자 대신 `color:var(--accent)` 텍스트 강조 또는 inset 라인으로 대체 검토(단 segmented control 관용상 현행도 허용 범위).
- **추천 카드 태그 정렬**: `tag--type`(쇼핑)·`tag--private`(프라이빗) 짝이 형태 동일·색만 차이로 사용자 원칙 부합(굿). 다만 `tag--private`만 앞에 도트(5px)가 붙어 두 태그 좌패딩이 다름(`tag--type` 9px vs `tag--private` 8px+도트). 짝 요소 통일 원칙상 도트 유무로 인한 시각 폭차 미세 조정.
- **At a Glance 상/하 패딩 비대칭**: `:nth-child(-n+2){padding-top:2px}` / `:nth-last-child(-n+2){padding-bottom:2px}`로 첫·끝 행만 2px — 의도된 엣지 정렬이나, 십자선 그리드에서 상단 셀이 살짝 떠 보일 수 있어 시각 검증 권장.
- **route-item 코드 칩 색**: SEL/ICN/GMP 칩이 바이올렛(`route-item__code{background:var(--private-soft);color:var(--private)}`) — 공항 코드는 이동/위치 정보라 코발트 계열이 더 직관적일 수 있으나, "프라이빗 동선"이라는 의미 부여면 현행 유지 가능. 일관성 차원 점검.
- **소수 라운드 미세 어긋남**: `bk__seg 12px / bk__seg button 9px / bk__input 11px / bk__counter 11px` — 9·11·12 혼재. 4px 그리드상 8·12로 정리하면 더 정돈됨.

## 결론

크림+코발트+바이올렛 톤은 정확히 "프리미엄 쇼퍼"를 향하고 있고, 면·여백·라인으로만 분리한 절제는 사용자 확정 원칙을 충실히 따른다. **상위 점수로 가는 단 두 가지 레버는 (1) 히어로를 풀폭·고높이·사진 위 카피로 키워 첫인상 임팩트를 회복하는 것, (2) 코발트=액션 / 바이올렛=프라이빗의 색 의미를 본문 상단에서 분명히 드러내 배분 불균형을 잡는 것이다.** 이 둘만 처리하면 8.0 → 8.7 수준의 완성도에 도달한다.
